# Спецификация приложения «Сборник песен» v2

## 1. Обзор

Мобильное PWA-приложение для поиска, просмотра и коллекционирования песен (гимнов). Основной пользователь — верующий, которому нужно быстро найти песню по номеру или тексту и петь. Приложение работает оффлайн, обновления скачиваются в фоне при наличии интернета.

**Ключевой принцип**: минимум кликов до нужной песни, текст легко читается во время пения.

---

## 2. Решения из интервью

### 2.1. Верстка куплетов и припевов

**Принято**: Вариант А «Лист песни» — единый фон на всю песню, без карточек/рамок для каждого куплета.

- Номер куплета (`1.`, `2.`) — цветная метка слева (цвет `--primary`)
- Метка припева — слово «Припев» слева, на уровне нумерации. Если позволяет ширина — полностью, если нет — перемещается над текстом припева. Нумерация припевов не отображается
- Куплеты разделены только отступами
- Максимальная читаемость при пении — достаточная высота строк, комфортный межстрочный интервал

Пример отображения:
```
1.  Слушайте повесть любви в простоте,
    Слушайте дивный рассказ:
    Бог нас навеки простил во Христе,
    Бог нас от гибели спас.

    Припев:
    Бог нас от гибели спас!
    Бог нас от гибели спас!

2.  Если неправда потерянных дней
    Мучит вас в тягостный час...
```

### 2.2. Повторы в тексте

Синтаксис повторов в исходном тексте:
- `/` — начало повтора
- `/Nр.` — конец повтора (N — количество раз: 2р., 3р. и т.д.)
- Повторы могут быть вложенными: несколько `/` подряд открывают разные уровни
- `/` без закрывающего `/Nр.` — ошибка в данных
- Модель парсинга — **стек**: каждый `/` добавляет уровень, `/Nр.` закрывает текущий

В UI: повторы подсвечиваются курсивом и цветом `--text-secondary`. Не разворачиваются.

### 2.3. Аккорды

- Формат: `{любой текст}` — отображается как аккорд
- `{Am}` — аккорд над текстом (chord-up)
- `{_G}` — инлайн-аккорд
- Свобода внутри скобок: `{(Am)}`, `{Am - в конце}`, `{Dm/C}`, `{D.S.}` — всё допустимо
- При `showChords=false` всё содержимое `{...}` удаляется из отображения
- Транспонирование: из начала содержимого `{}` извлекается нота и смещается на N полутонов
- Если аккордов у песни нет — кнопки транспонирования не показываются

### 2.4. Транспонирование аккордов

- Диапазон: от -6 до +5 полутонов (как в гитарных приложениях)
- Хранение: отдельная таблица `songSettings` в IndexedDB с полями `songNumber`, `transpose` (default: 0)
- Кнопки +/- на странице песни, сохраняются сразу в БД
- Не хранится в подборке (принято решение: смешение ответственности)
- Не показывается, если у песни нет аккордов

### 2.5. Боковое меню (гамбургер)

Структура (сверху вниз):
```
Меню                                    [X]

Главная
---
Подборки                               [+]
  ⭐ Избранное (3)
  Сборная (5)
  Свадьба (12)
---
Настройки
```

- «Избранное» — первое в списке со звёздочкой, неудаляемое, не переименовываемое, создаётся автоматически при первом запуске
- Подборки визуально обособлены разделителями и заголовком «Подборки»
- Кнопка создания подборки `[+]` — рядом с заголовком «Подборки»
- Каждая подборка показывает количество песен
- Сортировка подборок: по дате создания (последние снизу)
- «Настройки» — в самом низу меню, после подборок
- Если подборок много — секция подборок скроллится

### 2.6. Страница подборки

- Стилизация в том же ключе, что и варианты поиска (номер + название + badge варианта)
- Кнопка «Редактировать» наверху страницы — переводит в режим редактирования:
  - У каждой песни появляется красная кнопка удаления (корзина/минус)
  - Переход по клику на песню отключается
  - Кнопка меняется на «Готово»
- Удаление подборки — на странице самой подборки, не в меню

### 2.7. Главная страница

- Поле поиска (autofocus при загрузке) — остаётся без изменений
- Под полем поиска — инструкция по использованию приложения
- Если избранное пустое — инструкция более развёрнутая
- Пользователь не возвращается на главную после начала работы — все нужные функции доступны из страницы песни и меню

### 2.8. «Не гасить экран»

- Опция в настройках (toggle)
- По умолчанию: **включено**
- Технически: `navigator.wakeLock.request('screen')`
- Приоритет: Android (Chrome/Edge). iOS — по возможности, не критично

### 2.9. Горизонтальный режим

- Текст песни: ограничить max-width для читаемости
- Навбар: без изменений (и так скрывается при скролле)
- Табы вариантов: без изменений
- Кнопки навигации: без изменений
- Актуальность ниже, чем вертикальный — стихотворные строфы роднее вертикальной ориентации

### 2.10. Автообновление базы песен

- При запуске приложения: в фоне проверять хеш/etag `songs.json`
- Если хеш отличается:
  - Toast-уведомление: «Доступно обновление текстов песен»
  - Бейдж на иконке настроек в навбаре
  - Пользователь сам запускает обновление (кнопка в настройках или из toast)
- Концепция: приложение работает без интернета, обновление не мешает пользователю
- Если обновление занимает <10 секунд — можно обновлять сразу при подтверждении, без сложной системы предзагрузки

## 3. Хранилище данных — новый источник истины

### 3.1. Отдельный репозиторий `songs-data`

- Содержит YAML-файлы с текстами песен
- Одна песня — один файл: `songs/0001.yaml`, `songs/0002.yaml`, ...
- Также содержит `sections.yaml` с тематическими разделами
- Редакторы: человек + AI/LLM
- При пуше в main — GitHub Action склеивает все YAML в один файл `all-songs.yaml`

### 3.2. Формат YAML-файла песни (вариант В — компромисс)

```yaml
n: 1
title: Слушайте повесть любви в простоте
variants:
  - label: ""
    parts:
      - verse: |
          Слушайте повесть любви в простоте,
          Слушайте дивный рассказ:
          Бог нас навеки простил во Христе,
          Бог нас от гибели спас.
      - chorus: |
          Бог нас от гибели спас!
          Бог нас от гибели спас!
          Да! Бог нас навеки простил во Христе,
          Бог нас от гибели спас!
      - verse: |
          Если неправда потерянных дней
          Мучит вас в тягостный час,
          Верьте всем сердцем и верою всей:
          Бог нас от гибели спас.
```

Мульти-вариантная песня:
```yaml
n: 235
title: Со Христом бодрее в путь пойду
variants:
  - label: а
    parts:
      - verse: |
          Со Христом бодрее в путь пойду я...
      - verse: |
          Через реки, чрез моря и горы...
  - label: б
    parts:
      - verse: |
          Со Христом бодрее в путь пойду я...
      - chorus: |
          Через реки, чрез моря и горы...
```

Песня с аккордами:
```yaml
n: 1
title: Слушайте повесть любви в простоте
variants:
  - label: ""
    parts:
      - verse: |
          {Am}Слушайте повесть {Dm}любви в простоте,
          {G}Слушайте дивный {C}рассказ:
          {Am}Бог нас навеки {Dm}простил во Христе,
          {G}Бог нас от {C}гибели спас.
```

Песня с повторами:
```yaml
n: 235
title: Со Христом бодрее в путь пойду
variants:
  - label: а
    parts:
      - verse: |
          /Со Христом бодрее в путь пойду я,
          Только с Ним я радостен душой.
          К цели вечной я стремлюсь, ликуя,
          Доведёт лишь Он меня домой./2р.
```

**Формат `sections.yaml`:**
```yaml
sections:
  - title: Перед началом собрания
    songs: [1, 2, 3, 4, 5]
  - title: Призыв к спасению
    songs: [6, 7, 8]
```

### 3.3. Парсинг и сборка

**В `songs-data` (CI/CD):**
- GitHub Action: при пуше в main
- Склеивает все `songs/*.yaml` в один `all-songs.yaml`
- Публикует артефакт

**В `nuxt-songs-app` (CI/CD):**
- GitHub Action: при сборке скачивает `all-songs.yaml` из `songs-data`
- Скрипт конвертации `all-songs.yaml` → `songs.json` в формате, ожидаемом приложением
- `songs-data` не знает о формате JSON — разделение ответственности

### 3.4. Миграция данных при обновлениях

Ключ связи — `songNumber` (номер песни). Номера не меняются, поэтому:
- Связи в `songCollections` не ломаются
- Настройки в `songSettings` (транспонирование) не ломаются
- Обновление `songs.json` через `addSongs()` перезаписывает содержимое песен (правки текстов, переносы, аккорды)
- Пользовательские данные (подборки, настройки) сохраняются

## 4. Схема IndexedDB (обновлённая)

БД `SongsDB`, версия 6.

### 4.1. songs (без изменений)
- `number` (keyPath) — номер песни
- `title` — название
- `variants` — массив вариантов с body

### 4.2. collections (без изменений)
- `id` (keyPath, autoIncrement)
- `name` — название подборки
- `isFavorite` — флаг системной подборки «Избранное» (индекс)
- `createdAt`, `updatedAt` — метки времени ISO

### 4.3. songCollections (без изменений)
- `id` (keyPath, autoIncrement)
- `collectionId` — ID подборки (индекс)
- `songNumber` — номер песни (индекс)
- `variantIndex` — индекс варианта (default: 0)
- `addedAt` — метка времени ISO
- Составной уникальный индекс `collectionId_songNumber_variantIndex`

### 4.4. songSettings (НОВОЕ)
- `songNumber` (keyPath) — номер песни
- `transpose` — транспонирование в полутонах (default: 0, диапазон: -6..+5)

## 5. Страницы и компоненты — спецификация

### 5.1. Layout (`layouts/default.vue`)

**Навбар (56px, fixed top):**
- Слева: гамбургер (открывает боковое меню)
- Центр: `#navbar-center` — Teleport-слот для заголовка страницы
- Справа: иконка настроек с бейджем при наличии обновления

**Боковое меню (280px, slide-in слева):**
- Заголовок «Меню» + кнопка закрытия
- Пункт «Главная» (иконка дома)
- Разделитель
- Секция «Подборки» с кнопкой `[+]`
  - ⭐ Избранное (N) — первая, системная
  - Остальные подборки по дате создания, с количеством песен
- Разделитель
- Пункт «Настройки» (иконка шестерни) — прижат к низу

**Smart navbar:** скрывается при скролле вниз, появляется при скролле вверх.

### 5.2. Главная (`pages/index.vue`)

- Полнотекстовый поиск (SongSearchInput) — autofocus
- Под полем поиска — инструкция по использованию
- При пустой БД — предложение обновить через настройки
- Внизу контента страницы — версия приложения: `v{версия} · {короткий хеш коммита} · {дата сборки}` (например: `v1.0.0 · abc1234 · 21.05.2026`)
  - Версия — из `package.json`
  - Хеш — короткий SHA текущего коммита (7 символов), подставляется через env-переменную при сборке (`COMMIT_SHA`)
  - Дата — дата сборки, подставляется через env-переменную (`BUILD_DATE`)
  - Доступно через `useRuntimeConfig().public`: `appVersion`, `appCommit`, `appBuildDate`
  - Стиль: мелкий шрифт, цвет `--text-secondary`, выравнивание по центру

### 5.3. Страница песни (`pages/song/[number].vue`)

**Навбар:** ← номер → (переход между песнями, popover для быстрого поиска)

**Контент:**
- Заголовок песни (h1) + звёздочка избранного
- SongDisplay с новым дизайном «Лист песни»
- Если есть аккорды: кнопки транспонирования (+/-) с текущим значением
- Чипы подборок внизу (без изменений)

### 5.4. SongDisplay (`components/SongDisplay.vue`) — редизайн

**Новый дизайн «Лист песни»:**
- Единый фон (без отдельных карточек для куплетов)
- Номер куплета — цветная метка (`--primary`) слева
- «Припев» — метка слева, на уровне нумерации. Адаптируется по ширине: если помещается — слева, если нет — над текстом
- Нумерация припевов не отображается
- Разделение между частями — отступ
- Повторы (`/.../Nр.`) — курсив + цвет `--text-secondary`
- Аккорды: `{текст}` — над строкой или инлайн (без изменений)
- Транспонирование: применяется при рендеринге аккордов

**Адаптивность:**
- max-width контента ограничен для горизонтального режима
- Размер шрифта из настроек (small/medium/large)

### 5.5. Страница подборки (`pages/collections/[id].vue`)

- Заголовок подборки + количество песен
- Список песен: номер + название + badge варианта (стиль как в результатах поиска)
- Кнопка «Редактировать» → режим редактирования:
  - Красная кнопка удаления у каждой песни
  - Переход по клику отключён
  - Кнопка «Готово» для выхода из режима
- Удаление подборки — кнопка на этой же странице

### 5.6. Настройки (`pages/settings.vue`)

- Тема: light/dark/system (SettingToggle)
- Размер шрифта: small/medium/large (SettingToggle)
- Отображение аккордов: toggle switch
- Не гасить экран: toggle switch (default: вкл)
- Обновление базы данных: кнопка + статус
- Бейдж на пункте обновления, если доступно обновление

## 6. CI/CD

### 6.1. Репозиторий `songs-data`

```
songs-data/
├── songs/
│   ├── 0001.yaml
│   ├── 0002.yaml
│   └── ...
├── sections.yaml
└── .github/workflows/build.yaml
```

**Workflow `build.yaml`:**
1. При пуше в main
2. Склеивает `songs/*.yaml` → `all-songs.yaml`
3. Публикует `all-songs.yaml` как артефакт/релиз

### 6.2. Репозиторий `nuxt-songs-app`

**Workflow (существующий + доработка):**
1. При пуше в main или по расписанию
2. Скачивает `all-songs.yaml` из `songs-data`
3. Конвертирует `all-songs.yaml` → `public/assets/songs.json`
4. `npm run generate`
5. Деплой на GitHub Pages

### Техдолг — Линтер songs-data
21. Удалить ремарки `[текст]` из поискового индекса (`lib/search.js`). Сейчас `cleanText()` снимает квадратные скобки, но текст ремарок попадает в индекс. Нужно добавить `text.replace(/\[[^\]]*\]/g, '')` в `prepareSongForIndexing` или `cleanText`

Нужна эвристика для отличия переносов от настоящих ошибок — например, анализ стихотворного метра из мета-блока `meter`: если строка короче полного размера, это перенос и строчная допустима
