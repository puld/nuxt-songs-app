## Архитектура системы

### Компоненты:
1. **Агент планирования** - Создает планы и сохраняет контекст
2. **Агент исполнения** - Выполняет задачи и создает отчеты
3. **Менеджер контекста** - Управляет сохранением и загрузкой контекста

### Поток работы:
1. Сессия планирования → Создание плана → Сохранение контекста
2. Сессия исполнения 1 → Выполнение задачи 1 → Обновление статуса
3. Сессия исполнения 2 → Выполнение задачи 2 → Обновление статуса
4. ... (повторяется для всех задач)
5. Итоговый отчет → Очистка контекста

### Преимущества:
- Разделение планирования и исполнения
- Сохранение контекста между сессиями
- Отслеживание прогресса
- Возможность восстановления после сбоев
- Параллельное выполнение независимых задач
- Проверяемость каждой задачи
