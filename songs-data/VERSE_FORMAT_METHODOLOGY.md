# Методология: перевод текстов песен в стихотворные строфы

## Цель

Преобразовать тексты куплетов и припевов из сплошной строки в стихотворные строфы с переносами строк в файлах `songs-data/songs/*.txt`.

## Исходные данные

1565 файлов формата `NNNN.txt` в `songs-data/songs/`.

### Два типа форматирования

**Тип A — уже с переносами** (напр. #1):
```
1. Слушайте повесть любви в простоте,
Слушайте дивный рассказ:
Бог нас навеки простил во Христе,
Бог нас от гибели спас.
```

**Тип B — сплошная строка** (напр. #4, #5, #12):
```
1. Господь! Душа внимать готова, Лишь слух и очи мне открой Услышать правду Божья Слова, Увидеть свет небесный Твой. Дай в душу Слова разуменье, А в Нём земных скорбей забвенье, А в Нём земных скорбей забвенье.
```

Задача — перевести Тип B в Тип A.

## Принцип разбиения

Разбиение выполняется LLM (не механическим скриптом) с использованием языкового понимания.

### Три маркера определения разрыва строки

1. **Заглавная буква** после строчной — сильный сигнал начала новой строки строфы
   - НО: заглавные в обращениях к Богу (`О Боже`, `Дух Святой`, `Отец`), имена собственные (`Иисус`, `Христос`) — **НЕ** являются маркерами переноса
2. **Рифма** — конец стихотворной строки. Рифменные паттерны: ABAB, AABB, ABABCC и т.д. Рифмующееся слово = конец строки
3. **Стихотворный размер (метр)** — строки одной строфы обычно одного размера (4-стопный ямб, 3-стопный амфибрахий и т.д.). Используется для уточнения разрывов

## Правила форматирования результата

| Элемент | Правило |
|---------|---------|
| Номер куплета | В начале первой строки: `1. Строка` |
| Разделитель строк строфы | `\n` (одиночный перенос) |
| Разделитель строф | `\n\n` (пустая строка) |
| Заглавные буквы | **Каждая строка строфы начинается с заглавной буквы.** Без исключений — даже предлоги, союзы, частицы («в», «и», «к», «с», «о», «на», «по» и т.д.) пишутся с заглавной буквы в начале строки |
| Отступы | **Без отступов** в начале строк (не выравнивать под номер) |
| Репризы (`/2р.`, `/3р.`) | Продолжение строки, как в исходнике. См. подробнее ниже |
| Аккорды (`{Am}`, `{_G}`) | Оставить как есть |
| Припевы | Разбиваются так же как куплеты |
| Заголовок песни | Не меняется (`#N название`) |
| Название песни | Должно совпадать с началом первого куплета (или припева, если песня начинается с припева). Как минимум — первая строка строфы. Если фраза обрывается (смысл не завершён) — включить часть или всю вторую строку. См. подробнее ниже |
| Маркеры вариантов | Не меняются (`(а)`, `(б)`, `(вариант для сестёр)`) |

### Правило реприз

Если строка (или несколько строк) повторяется в исходнике дословно — это реприза. Вместо дублирования текста используется формат репризы: повторяемый текст оборачивается в `/.../`, после чего ставится маркер кратности (`2р.`, `3р.`).

**Формат:** `/повторяемый текст/ Nр.`

**Правила:**
- Повторяемый текст пишется один раз, обёрнутый в слеши `/.../`
- После закрывающего слеша — пробел и маркер кратности (`2р.`, `3р.` и т.д.)
- Реприза — продолжение строки (на той же строке, что и предшествующий текст, через пробел, или на отдельной если того требует формат)
- Если повторяются несколько строк подряд — все они заключаются в одни `/.../`
- **Текст после репризы** — если в исходнике после `/.../ Nр.` идёт продолжение текста, решение о переносе на новую строку принимается по контексту строфы (размер, рифма, смысловая цельность). Не механическое правило, а анализ конкретного случая

**Примеры:**

Исходник с дублированием:
```
А в Нём земных скорбей забвенье,
А в Нём земных скорбей забвенье.
```
Результат:
```
/А в Нём земных скорбей забвенье./ 2р.
```

Исходник с репризой и продолжением — текст после репризы переносится (контекст: новая строка строфы):
```
/Дай слово им Твоё принять /2р. и мир вкусить святой опять.
```
Результат:
```
/Дай слово им Твоё принять /2р.
И мир вкусить святой опять.
```

Исходник с репризой без продолжения:
```
/Твою волю во всём исполнять. /2р.
```

### Правило названия песни

Название песни в заголовке `#N название` должно быть **идентично** началу текста первого куплета (или первого припева, если песня начинается с припева). Правила:

1. **Минимум** — первая строка строфы первого куплета
2. **Если фраза обрывается** — включить часть или всю вторую строку, чтобы название было осмысленным
3. **Знаки препинания** в конце названия не включаются (запятая, точка с запятой, двоеточие), но восклицательный и вопросительный знаки сохраняются
4. **Знаки внутри названия** (запятая между частями фразы, тире, дефис) сохраняются

Примеры:

| Первая строка строфы | Вторая строка | Название | Комментарий |
|---|---|---|---|
| `Слушайте повесть любви в простоте,` | `Слушайте дивный рассказ:` | `Слушайте повесть любви в простоте` | Фраза завершена, запятая в конце убрана |
| `Господь! Душа внимать готова,` | `Лишь слух и очи мне открой` | `Господь! Душа внимать готова` | Фраза завершена смыслом |
| `О Спаситель! Благодать` | `На благую весть излей;` | `О Спаситель! Благодать На благую весть` | Фраза обрывается — включена вторая строка |
| `Благословений потоки Бог обещал ниспослать,` | `Ближним, чужим и далёким...` | `Благословений потоки Бог обещал ниспослать` | Фраза завершена |
| `Ах куда сокроюсь я?` | `В ком и где искать спасенье?` | `Ах куда сокроюсь я?` | Вопросительный знак сохранён |

**Важно:** при разбиении сплошного текста на строфы, нужно проверять что название в заголовке совпадает с получившейся первой строкой. Если не совпадает — либо название нужно исправить, либо разбиение нужно пересмотреть. Отмечать такие случаи для ревью.

## Метаинформация (блок @meta)

Анализ песни (метр, рифмовка, заметки о разрывах строк) сохраняется прямо в `.txt` файле в виде блока `@meta`/`@end` после заголовка песни. Парсер `parse.js` пропускает этот блок — он не попадает в итоговый `songs.json`.

### Формат

```
#4 Господь! Душа внимать готова

@meta
meter: 4-стопный ямб
rhyme: ABABCC
note: «Божья» — прилагательное внутри строки, не переносить
@end

1. Господь! Душа внимать готова,
Лишь слух и очи мне открой
...
```

### Правила мета-блока

| Правило | Описание |
|---------|----------|
| Расположение | Сразу после заголовка песни и пустой строки, перед первым куплетом |
| Маркер начала | `@meta` на отдельной строке |
| Маркер конца | `@end` на отдельной строке |
| Пустая строка | После `@end` — обязательна (отделяет мета-блок от тела песни) |
| Поля | `meter:`, `rhyme:`, `note:` — каждое на своей строке |
| `meter` | Стихотворный размер (4-стопный ямб, 3-стопный амфибрахий и т.д.) |
| `rhyme` | Рифмовка строфы (ABAB, AABB, ABABCC и т.д.) |
| `note` | Заметки о неочевидных разрывах, можно несколько строк `note:` |
| Опциональность | Блок `@meta` — опциональный. Если нет — парсер работает как раньше |

### Примеры с мета-блоком

#### Простой случай
```
#1 Слушайте повесть любви в простоте

@meta
meter: 4-стопный хорей
rhyme: ABAB
@end

1. Слушайте повесть любви в простоте,
Слушайте дивный рассказ:
Бог нас навеки простил во Христе,
Бог нас от гибели спас.
```

#### С заметками
```
#4 Господь! Душа внимать готова

@meta
meter: 4-стопный ямб
rhyme: ABABCC
note: «Божья» — прилагательное внутри строки, не начало новой строки
@end

1. Господь! Душа внимать готова,
Лишь слух и очи мне открой
Услышать правду Божья Слова,
Увидеть свет небесный Твой.
Дай в душу Слова разуменье,
А в Нём земных скорбей забвенье,
А в Нём земных скорбей забвенье.
```

#### С несколькими заметками
```
#12 Открой сердца детей Своих

@meta
meter: 4-стопный хорей
rhyme: AABB
note: «О Боже» — обращение, не переносить
note: реприза /2р. — продолжение строки строфы
@end

1. Открой сердца детей Своих
И Сам войди, О Боже, в них!
Дай слово им Твоё принять
И мир вкусить святой опять!
/Дай слово им Твоё принять /2р. и мир вкусить святой опять.
```

### Изменение парсера

Парсер `parse.js` нужно дополнить: в функции `parseSongFile` перед обработкой тела песни пропускать блок `@meta`...`@end`. Строки между этими маркерами игнорируются и не попадают в выходной JSON.

## Рабочий процесс

1. Читаю пачку из 5-10 файлов `.txt`
2. Определяю какие куплеты/припевы — сплошной текст (Тип B), а какие уже разбиты (Тип A)
3. Для Типа B — разбиваю текст на стихотворные строки, используя языковое понимание (рифма + размер + заглавные)
4. Для Типа A — удаляю отступы в начале строк, если есть
5. Показываю **diff** для каждого изменённого файла
6. Жду подтверждения / правок
7. Записываю одобренные изменения

## Неоднозначные случаи

Помечаю в diff места где я не уверен (например, строка может разрываться по-разному в зависимости от трактовки размера). Решение принимает ревьюер.

## Примеры

### Пример 1: Песня #4 (Тип B → Тип A)

**Было:**
```
1. Господь! Душа внимать готова, Лишь слух и очи мне открой Услышать правду Божья Слова, Увидеть свет небесный Твой. Дай в душу Слова разуменье, А в Нём земных скорбей забвенье, А в Нём земных скорбей забвенье.
```

**Стало:**
```
1. Господь! Душа внимать готова,
Лишь слух и очи мне открой
Услышать правду Божья Слова,
Увидеть свет небесный Твой.
Дай в душу Слова разуменье,
А в Нём земных скорбей забвенье,
А в Нём земных скорбей забвенье.
```

Анализ: 4-стопный ямб, рифмовка ABABCC. «Божья» — не начало строки, а прилагательное внутри строки.

### Пример 2: Песня #5 (Тип B → Тип A)

**Было:**
```
1. О Спаситель! Благодать На благую весть излей; Дай нам все слова понять, Все слова любви Твоей!
```

**Стало:**
```
1. О Спаситель! Благодать
На благую весть излей;
Дай нам все слова понять,
Все слова любви Твоей!
```

Анализ: 4-строчная строфа, рифмовка ABAB. «На» после «Благодать» — начало новой строки (заглавная), «Дай» — тоже. «Все» — заглавная = новая строка.

### Пример 3: Реприза (песня #12)

**Было:**
```
1. Открой сердца детей Своих И Сам войди, О Боже, в них! Дай слово им Твоё принять И мир вкусить святой опять! /Дай слово им Твоё принять /2р. и мир вкусить святой опять.
```

**Стало:**
```
1. Открой сердца детей Своих
И Сам войди, О Боже, в них!
Дай слово им Твоё принять
И мир вкусить святой опять!
/Дай слово им Твоё принять /2р. и мир вкусить святой опять.
```

Анализ: «И Сам» — заглавная, начало строки. «О Боже» — обращение, НЕ начало новой строки. Реприза — продолжение последней строки строфы.

## История изменений методологии

- Начальная версия — сформулирована по итогам интервью
- Добавлен формат мета-блока `@meta/@end` для сохранения анализа прямо в .txt файлах
- Добавлено правило названия песни: должно совпадать с началом первого куплета/припева
- Добавлено правило реприз: дублирующиеся строки заменяются на `/текст/ Nр.`
- Добавлено правило заглавных букв: каждая строка строфы начинается с заглавной буквы без исключений
